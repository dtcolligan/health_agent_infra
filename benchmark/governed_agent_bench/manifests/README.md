# Frozen Manifests

This directory stores runtime contract snapshots used by benchmark
tasks. The current reference snapshot is `hai_0_2_0.json`, generated
from:

```bash
uv run hai capabilities --json
```

Snapshots are committed as provenance envelopes, not raw command dumps.
The envelope records the source commit, generation command, HAI version,
contract schema version, and embedded manifest. Stability checks compare
the embedded manifest while ignoring `generated_at`.

`hai_0_2_0.json` was generated from source commit
`0f114e31da67bcd245702c903d8e12b0bb92a197`, with embedded
`contract_schema_version` `agent_cli_contract.v2`.

`agent_cli_contract_v1_drift.json` is the deliberately stale L7 drift snapshot.
It is generated by:

```bash
uv run python benchmark/governed_agent_bench/manifests/build_stale_manifest_snapshot.py
```

The source checkout has no `v0.1.18` tag, so the generator anchors to
release-prep commit `aa418d5e8fefca86f2d2a66f19975e63e31473c5` and
the committed v1 manifest snapshot at
`verification/tests/snapshots/cli_capabilities_v0_1_13.json`. The
embedded manifest remains `agent_cli_contract.v1` and reports
`hai_version` `0.1.17`; the benchmark-level snapshot id therefore names
the contract shape (`agent_cli_contract_v1_drift`) rather than a HAI
release version. Benchmark harnesses must treat it as stale contract
input, not as a runtime downgrade.

Do not replace a manifest in place after tasks depend on it. Add a new
snapshot and use L7 drift tasks to compare behavior across versions.
